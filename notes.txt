Good question—this is exactly the kind of thing panelists like to test: whether you actually understand what you built, not just that it works.

Here’s a clear, confident way to answer.

🎤 Simple Answer (what you say in defense)

“We used a combination of classical computer vision algorithms from OpenCV. For face detection, we used the Haar Cascade classifier, and for face recognition, we used the LBPH (Local Binary Pattern Histogram) algorithm. For person detection, we used OpenCV’s pre-trained HOG (Histogram of Oriented Gradients) with a linear SVM classifier.”

🧠 If they ask “explain it more”
👤 Face Detection (Haar Cascade)

“We used Haar Cascade, which detects faces by scanning the image for specific patterns like edges, eyes, and facial structures using features learned from positive and negative samples. It works in real time and is lightweight.”

✔ Keyword: feature-based object detection

🧑 Face Recognition (LBPH)

“For recognizing the person, we used LBPH. It converts the face into a texture-based representation using local binary patterns and compares it with trained faces using histogram distance. Lower confidence means better match.”

✔ Keyword: texture-based recognition

🚶 Person Detection (HOG + SVM)

“For detecting a full human body, we used HOG combined with a Support Vector Machine. HOG extracts gradient orientation features from the image, and the SVM classifies whether a human is present based on a pre-trained model.”

✔ Keyword: shape + gradient-based detection

⚡ One-line technical summary (very strong for panel)

“Our system uses Haar Cascade for face detection, LBPH for face recognition, and HOG + SVM for full-body person detection, all from OpenCV’s pre-trained classical computer vision models.”

🔥 If they try to challenge you
❓ “Is this AI or machine learning?”

You answer:

“LBPH and SVM are machine learning-based methods, while Haar Cascade is a feature-based classical computer vision technique. So our system is a hybrid of classical CV and machine learning.”

❓ “Why not deep learning like YOLO?”

“We chose lightweight classical models because they are faster, require no GPU, and are suitable for Raspberry Pi deployment with real-time performance.”

🧠 Bonus (makes you sound advanced)

You can add:

“The system is optimized for edge computing, so we prioritized real-time performance over heavy deep learning models.”

🏁 Final Panel-Ready Answer (memorize this)

“We used Haar Cascade for face detection, LBPH for face recognition, and HOG with SVM for person detection. These are OpenCV-based classical computer vision and machine learning algorithms optimized for real-time edge device processing like Raspberry Pi.”
